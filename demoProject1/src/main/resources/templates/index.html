Spring Boot

src/main/resources의 templates 혹은 static에 index.html 이 존재하면 메인 페이지 역할을 해줌(resolver(Spring Boot 내장 기능)가 index.html 파일을 찾음)

Spring Boot Project 실행 방법 (서버 배포 방법)

        src/main/java 하위 패키지에 
        XXXApplication.java 파일이 프로젝트 생성 시 자동 생성됨
        (XXX == 프로젝트명)
        
        해당 클래스에는 main 메서드가 존재
        -> 실행 -> 내장된 Tomcat Server가 배포 시작

        (* 브라우저가 자동으로 실행되지 않기 때문에 직접 브라우저 켜서 주소 입력해야함)

프로젝트 구조

  • src/main/java : 자바 소스 코드를 모아두는 폴더
  • src/main/resources : 자바에서 사용할 자원, 설정 파일을 모아두는 폴더
    • templates : Java 템플릿(응답화면)을 모아두는 폴더(.html 확장자 : thymeleaf)
    • static : templates에 적용될 정적 자원을 모아두는 폴더(JS, CSS, 이미지)
    • application.properties : Spring Boot Project 관련된 전반적 설정을 정의하는 파일(중요!!!)
  • Project and External Dependencies : build.gradle에 명시된 라이브러리 모음
  • build.gradle : gradle(빌드 도구) 프로젝트에 필요한 라이브러리, 배포, 빌드 관련 설정을 작성하는 파일
  1. 요청 + forward 처리
  2. 정적 자원(JS, CSS, 이미지) 연결
  3. 서버로 파라미터 제출
